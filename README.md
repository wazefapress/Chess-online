# Chess-online
اللعب مع الكمبيوتر و أونلاين 
https://wazefapress.github.io/Chess-online/

بناءً على الأكواد والمكتبات التي أستخدمت في تطوير هذه اللعبة طوال خطوات العمل، تتكون حزمة التقنيات والبرمجيات المستخدمة من الآتي:
• HTML5: لهيكل وصفحة العرض الرئيسية (index.html).
• CSS3: لتنسيق الواجهات، الخلفيات، التدرجات اللونية، وتأمين التجاوب الكامل مع جميع الشاشات.
• JavaScript (Vanilla JS): للتحكم بمنطق اللعبة، التوقيت، وإدارة الحالات برمجياً.
• jQuery: لتسهيل التعامل مع عناصر DOM، أحداث الأزرار، وتحديث الواجهات بشكل سريع وفعّال.
• Chess.js: المكتبة المسؤولة عن القواعد البرمجية للعبة الشطرنج (حساب الحركات المتاحة، فحص الكش، وكش مات).
• Chessboard.js: مكتبة واجهة المستخدم الخاصة برقعة الشطرنج لعرض القطع ودعم خاصية السحب والإفلات (Drag and Drop).
• Howler.js: مكتبة متخصصة لإدارة وتشغيل المؤثرات الصوتية بسلاسة (مثل صوت تحريك القطعة ونهاية اللعبة).
• Canvas Confetti: مكتبة لإضافة تأثيرات الاحتفال والقصاصات الملونة (Confetti) عند فوز أحد اللاعبين.
لتحقيق نمطي اللعب (اللعب ضد الكمبيوتر واللعب الجماعي مع لاعب حقيقي عبر الإنترنت)، تم الاعتماد على التقنيات والآليات التالية:
1. اللعب ضد الكمبيوتر (Player vs AI)
• المحرك المنطقي (Chess.js): يقوم بفحص صحة كل حركة ينفذها اللاعب، والتحقق من القواعد (مثل حالات الكش، كش مات، أو التعادل).
• خوارزمية الذكاء الاصطناعي (AI Logic): يتم توليد الحركات المتاحة للكمبيوتر باستخدام الدوال البرمجية للمكتبة، حيث يقوم النظام (في المستوى الأساسي أو المتوسط) باختيار حركة عشوائية صحيحة أو حسابية من قائمة الحركات القانونية المتاحة للطرف الآخر فور إتمام اللاعب لحرمنيته.
2. اللعب مع لاعب حقيقي عبر الإنترنت (Online Multiplayer)
• نظام الأكواد الفريدة (Room Codes): يعتمد على إنشاء معرف فريد لكل جلسة لعب (Room) لربط لاعبين ببعضهما البعض.
• آلية المزامنة والاتصال (Real-time Communication): تتطلب هذه الميزة عادةً خادم وسيط (مثل WebSocket أو خدمات مثل Firebase / Socket.io) لنقل حالة رقعة الشطرنج والحركات المتبادلة بين الجهازين بشكل لحظي، بحيث يتم تحديث موقع القطع عند الطرف الثاني فور إرسالها من الطرف الأول.
هذه اللعبة تعمل بالكامل على استضافة ملفات عادية (Static Hosting) مثل GitHub Pages أو Vercel أو Netlify أو أي استضافة ويب تقليدية، ولا تتطلب سيرفر خاصاً معقداً (مثل Node.js خادم دائم التشغيل) للجزء الخاص بواجهة اللعب واللعب ضد الكمبيوتر.

وإليك التفصيل حسب نمط اللعب:
1. اللعب ضد الكمبيوتر (Player vs AI) والواجهات:
◦ لا تتطلب سوى رفع ملفات (index.html, style.css, script.js) بالإضافة إلى ملفات المكتبات (مثل chess.js و chessboard.js وغيرها، والتي يمكن جلبها عبر روابط CDN مباشرة).
◦ كل العمليات الحسابية ومنطق اللعبة يتم معالجتها محلياً داخل متصفح المستخدم (Client-side)، مما يعني أن أي استضافة مجانية أو عادية للملفات الثابتة تكفي لتشغيلها بكفاءة تامة.
2. اللعب الجماعي أونلاين (Online Multiplayer):
◦ إذا كان الاعتماد على خادم وسيط (مثل WebSocket أو Node.js/Socket.io) لمزامنة الحركات بين لاعبين في غرف مختلفة: فهذا الجزء يتطلب سيرفر تشغيل (Backend Server) يستقبل الرسائل ويبثها بين الطرفين لحظياً.
◦ البديل بدون سيرفر دائم: إذا تم بناء ميزة الأونلاين باستخدام خدمات جاهزة بدون خادم خاص (Serverless) مثل Firebase Realtime Database أو خدمات مزامنة تعتمد على السحاب، فستبقى ملفات اللعبة وحدها على استضافة عادية، بينما يتم إدارة بيانات الغرف عبر تلك الخدمة السحابية الخارجية بكل سهولة.
